This repository provides extended hosts and block lists that can be used freely with Pi-hole, DNS filters, and other ad blocking systems.

The goal of this project is to go beyond standard blocklists by adding additional tracking, advertising, and telemetry domains while keeping everything simple, clean, and community driven.

## Purpose

- Extend default Pi-hole block capabilities
- Focus on ads, tracking, and telemetry domains
- Easy integration with Pi-hole or similar tools
- Community maintained and continuously improved

## Usage

You can add these lists directly into Pi-hole under Group Management -> Adlists or use them as a hosts file.

Example format:

0.0.0.0 example.com

## Contributing

Contributions are welcome.

You can help by:
- Adding new tracking or advertising domains
- Organizing lists into categories such as ads, tracking, social
- Reporting false positives
- Improving structure or performance

## How to contribute

1. Fork the repository
2. Add your domains to the appropriate list
3. Create a pull request
4. Describe what you added and why

## Guidelines

- Only include domains that clearly serve ads, tracking, or telemetry purposes
- Avoid personal or private blocklists
- Provide context or sources when possible
- Avoid duplicates

## Community

The quality of this list improves with contributions. The goal is to build a reliable, up to date, and community driven privacy blocklist for everyone.
